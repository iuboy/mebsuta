# 版本规范

本项目遵循 [语义化版本 2.0.0](https://semver.org/lang/zh-CN/) 规范。

## 版本格式

版本号格式：`v主版本号.次版本号.修订号`

- **主版本号 (MAJOR)**：不兼容的API变更
- **次版本号 (MINOR)**：向下兼容的功能新增
- **修订号 (PATCH)**：向下兼容的问题修复

## 版本策略

### 稳定版本 (v1.0.0+)

- 主版本 `v1.x.x` 表示稳定的公共API
- 承诺在主版本内保持向后兼容
- 不会引入破坏性变更

### 预发布版本

- `v0.x.x` 表示开发中的版本
- API可能随时变化
- 不保证向后兼容性

### 预发布标识符

- `v1.0.0-alpha`：Alpha版本，功能开发中
- `v1.0.0-beta`：Beta版本，功能基本完成，测试中
- `v1.0.0-rc.1`：Release Candidate，候选发布版本

## 版本标签

### 创建版本标签

```bash
# 创建补丁版本
git tag -a v1.0.1 -m "修复日志同步问题"

# 创建次版本
git tag -a v1.1.0 -m "添加Syslog输出支持"

# 创建主版本
git tag -a v2.0.0 -m "重构配置API，不兼容v1.x"

# 推送标签
git push origin v1.0.1
```

### 标签命名规范

- 使用 `v` 前缀
- 使用 `-` 分隔预发布标识符
- 使用 `+` 分离构建元数据（可选）

```
v1.0.0
v1.0.0-alpha
v1.0.0-beta.1
v1.0.0-rc.1
v1.0.0+build.123
```

## 发布清单

在发布新版本前，请确认：

- [ ] 所有测试通过
- [ ] 更新 CHANGELOG.md
- [ ] 更新模块版本号（go.mod不需要显式版本）
- [ ] 创建版本标签
- [ ] 推送标签到远程仓库
- [ ] 创建GitHub Release

## API 兼容性承诺

### v1.x.x 承诺

以下变更被视为破坏性变更，需要主版本号升级：

1. 删除或重命名导出的类型、函数、常量
2. 修改导出类型的签名
3. 修改函数签名的参数类型或返回值类型
4. 移除支持的配置选项

以下变更**不是**破坏性变更：

1. 添加新的导出类型、函数、常量
2. 扩展现有类型（添加新字段）
3. 添加新的配置选项
4. 性能优化（不改变API）
5. Bug修复

### 弃用策略

当一个API需要被弃用时：

1. 在文档中标记为 `Deprecated`
2. 在注释中说明替代方案
3. 保留至少一个次版本周期
4. 在下一个主版本中移除

```go
// Deprecated: 使用 NewLoggerConfigBuilder 替代，将在 v2.0.0 移除
func NewSimpleConfig(name string) *LoggerConfig {
    // ...
}
```

## 模块版本要求

本项目最低需要 Go 1.22 版本。

```go
// go.mod
module github.com/iuboy/mebsuta

go 1.22
```

## 依赖管理

### 直接依赖

生产依赖应保持稳定版本：
- 优先使用稳定的 release 版本
- 避免使用 pre-release 版本
- 定期更新依赖以获取安全修复

### 开发依赖

开发依赖（如测试工具）版本要求较宽松：
- testcontainers-go 需要 Go 1.21+
- 可以使用较新的版本获取新功能

### 更新依赖

```bash
# 更新所有直接依赖到最新版本
go get -u ./...

# 更新特定依赖
go get -u github.com/influxdata/influxdb-client-go/v2

# 清理未使用的依赖
go mod tidy
```
